¿Para que sirve HTML?

Es la base de toda página web, se encarga de estructurar el contenido.

  • por qué aprenderlo: sin HTML no puedes crear sitios web, ya que define títulos, párrafos, listas, imágenes, enlaces, formularios, etc.

Ventajas:

  • Es sencillo de aprender y entender.
  • Está estandarizado y soportado por todos los navegadores.
  • Permite crear la semántica del contenido.

Desventajas:

  • No es un lenguaje de programación (no tiene lógica ni condicionales).
  • Por sí solo, el diseño es muy limitado

¿Por qué debo aprender CSS?

Es el lenguaje de estilos que da formato al HTML.

  • por qué aprenderlo: controla colores, tamaños, fuentes, márgenes, posiciones y hace posible que un sitio luzca atractivo.

Ventajas:

  • Permite separar el contenido (HTML) del diseño
  • Ofrece control total sobre el aspecto visual.
  • Compatible con animaciones y diseños responsivos.

Desventajas:

  • Puede ser complejo en proyectos grandes.
  • No maneja lógica (solo presentación).

¿Entonces cómo funciona JS?

Es el lenguaje que da interactividad a las páginas web.

  • por qué aprenderlo: hoy en día es imprescindible en el desarrollo web, tanto en frontend como en backend (Node.js).

Ventajas:

  • Versátil: funciona en el navegador y en servidores.
  • Gran ecosistema de librerías y frameworks (React, Angular, Vue, etc.).
  • Alta demanda laboral.

Desventajas:

  • Puede volverse complejo de mantener en proyectos grandes sin buenas prácticas.
  • Su flexibilidad puede causar errores si no se programa con cuidado.

¿Java se sigue usando?

Lenguaje de propósito general, muy usado en aplicaciones empresariales.

  • por qué aprenderlo: es uno de los lenguajes más estables y usados en bancos, seguros, Android y sistemas grandes.

Ventajas:

  • Orientado a objetos, robusto y seguro.
  • Portabilidad: "Escribe una vez, ejecuta en cualquier lugar" gracias a la JVM.
  • Gran comunidad y soporte.

Desventajas:

  • Verboso: escribir programas simples puede requerir mucho código.
  • Puede ser más lento que lenguajes nativos como C o C++.

¿Saber Python es saber programar?

Lenguaje multipropósito conocido por su simplicidad y potencia.

  • por qué aprenderlo: es ideal para principiantes y expertos; se usa en ciencia de datos, IA, web y automatización.

Ventajas:

  • Sintaxis sencilla y muy legible.
  • Gran cantidad de librerías para diferentes áreas (Django, TensorFlow, Pandas, etc.).
  • Comunidad activa y creciente.

Desventajas:

  • No es tan rápido como C, C++ o Java.
  • El manejo de memoria no es tan eficiente para sistemas muy grandes.

¿Siempren mencionan MySQL, es un lenguaje de programación?

Sistema de gestión de bases de datos relacional (RDBMS) de código abierto muy popular en aplicaciones web. MySQL almacena datos en tablas y utiliza SQL para consultas, transacciones y administración de esquemas. Es conocido por su rendimiento, escalabilidad y amplia compatibilidad con lenguajes y plataformas (por ejemplo: PHP, Node.js, Java).

  • por qué aprenderlo: la mayoría de sistemas trabajan con datos, y SQL es clave para consultarlos, modificarlos y administrarlos.

Ventajas:

  • Potente para trabajar con grandes volúmenes de datos.
  • Estándar en bases de datos relacionales.
  • Permite consultas muy complejas con relativamente poco código.

Desventajas:

  • No es un lenguaje de propósito general (sirve solo para bases de datos).
  • La optimización de consultas puede ser complicada.

¿MongoDB es lo mismo que MySQL?

MongoDB es una base de datos orientada a documentos que guarda información en documentos BSON (similar a JSON). Organiza los datos en colecciones y documentos en lugar de tablas y filas, facilitando el manejo de datos semi-estructurados y el desarrollo ágil.

  • por qué aprenderlo: es muy útil cuando trabajas con datos cambiantes o modelos que evolucionan, APIs basadas en JSON, aplicaciones web y sistemas que requieren alta escalabilidad horizontal.

Ventajas:

  • Esquema flexible: permite almacenar documentos con estructuras diferentes en la misma colección, acelerando el desarrollo y la iteración.
  • Escalabilidad y alta disponibilidad: soporta sharding para distribuir datos y replica sets para replicación y failover automático.
  • Potente motor de consultas y agregación: ofrece índices, pipelines de agregación, consultas geoespaciales y operaciones sobre documentos tipo JSON.

Desventajas:

  • Mayor uso de almacenamiento por el formato BSON y, en algunos casos, menor eficiencia en consultas relacionales complejas frente a un RDBMS.
  • Al ser flexible, la falta de un esquema rígido puede llevar a inconsistencias si no se aplican buenas prácticas y validaciones a nivel de aplicación o mediante esquemas de validación de MongoDB.
